# IDDO-SDTM Tutorial Healthcare Encounters (HO) Domain

A fictional HO dataset for testing code and training

## Usage

``` r
HO_RPTESTB
```

## Format

### 

A data frame with 3 rows and 24 columns:

- STUDYID:

  Study Identifier

- DOMAIN:

  Domain Abbreviation

- USUBJID:

  Unique Subject Identifier

- HOSEQ:

  Sequence Number

- HOTERM:

  Reported Term for the Disposition Event

- HODECOD:

  StandardiSed Disposition Term

- HOPRESP:

  Pre-Specified Healthcare Encounter

- HOOCCUR:

  Healthcare Encounter Occurrence

- HOSTAT:

  Completion Status

- HOREASND:

  Reason Healthcare Encounter Not Done

- HODTC:

  Date/Time of Specimen Collection

- HOSTDTC:

  Start Date/Time of Disposition Event

- HOENDTC:

  End Date/Time of Healthcare Encounter

- HODY:

  Study Day of Specimen Collection

- HOSTDY:

  Study Day of Start of Disposition Event

- HOENDY:

  Study Day of End of Healthcare Encounter

- HODUR:

  Duration of Healthcare Encounter

- HOSTRF:

  Start Relative to Reference Period

- HOEVINTX:

  Evaluation Interval Text

- HOCDSTDY:

  Contributor-Defined Study Day

- HODISOUT:

  Discharge Outcome

- SELFCARE:

  Ability to Self-Care at Discharge

- HOINDC:

  Indication

- HOPATT:

  Number of Healthcare Encounters
